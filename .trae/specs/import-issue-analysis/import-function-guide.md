# 导入功能使用指南

## 概述
导入功能允许用户从备份文件中恢复账号和设置数据。本文档提供了导入功能的使用方法、常见问题和故障排除指南。

## 使用方法

### 1. 准备备份文件
- 确保备份文件是有效的 JSON 格式
- 备份文件应包含 `timestamp` 字段
- 备份文件可以包含账号数据、设置数据或两者都包含

### 2. 执行导入操作
- 在系统界面中找到导入功能入口
- 选择要导入的备份文件
- 点击"导入"按钮开始导入过程
- 等待导入完成并查看导入结果

### 3. 查看导入结果
- 导入完成后，系统会显示导入结果摘要
- 查看导入的账号数量、设置数量等统计信息
- 检查是否有错误或警告信息

## 常见问题及解决方案

### 问题：导入提示"账号 未导入，设置 未导入"
**原因**：
- 备份文件格式不正确
- 备份文件中没有可识别的账号或设置数据
- 备份文件缺少必要的字段

**解决方案**：
- 确保备份文件是有效的 JSON 格式
- 检查备份文件是否包含 `timestamp` 字段
- 确保备份文件中包含有效的账号或设置数据
- 尝试使用系统导出的备份文件进行导入

### 问题：导入过程中出现超时错误
**原因**：
- 备份文件过大
- 系统资源不足
- 网络连接问题

**解决方案**：
- 尝试使用较小的备份文件
- 确保系统有足够的内存和 CPU 资源
- 检查网络连接是否稳定
- 考虑分批次导入数据

### 问题：导入后部分数据丢失
**原因**：
- 备份文件中的数据格式不正确
- 导入过程中出现错误
- 系统版本不兼容

**解决方案**：
- 确保备份文件是由相同或兼容版本的系统导出的
- 检查备份文件中的数据格式是否正确
- 查看导入过程中的错误信息
- 尝试重新导出和导入数据

## 故障排除

### 检查导入日志
- 系统会记录导入过程的详细日志
- 日志位置：系统日志目录中的 `backup.log` 文件
- 查看日志可以了解导入过程中的具体错误信息

### 验证备份文件
- 使用 JSON 验证工具检查备份文件的格式
- 确保备份文件包含必要的字段：`timestamp`
- 检查备份文件的大小是否合理

### 测试导入功能
- 使用系统导出的备份文件进行测试
- 尝试导入不同类型的备份文件（只包含账号、只包含设置、两者都包含）
- 测试大型备份文件的导入性能

## 最佳实践

### 备份文件管理
- 定期创建备份文件
- 存储备份文件到安全的位置
- 备份文件命名应包含日期和时间信息
- 保留多个版本的备份文件

### 导入操作
- 在导入前备份当前系统数据
- 确保系统有足够的资源处理导入操作
- 导入后验证数据是否正确恢复
- 如遇到问题，查看错误信息并参考本文档的故障排除部分

## 技术细节

### 备份文件格式
- 版本：2.1
- 支持的类型：
  - `all`：包含账号和设置数据
  - `accounts`：只包含账号数据
  - `preferences`：只包含设置数据

### 导入过程
1. 验证备份文件格式
2. 检测账号和设置数据
3. 执行导入操作
4. 清理不再存在的记录
5. 插入辅助数据
6. 返回导入结果

### 错误处理
- 导入过程中的错误会被记录并显示给用户
- 部分错误不会导致整个导入过程失败
- 详细的错误信息有助于诊断问题

# 长时间运行操作测试计划

## 测试目标

验证系统能够处理长时间运行的操作，特别是备份导入功能，确保在处理大型数据集时不会出现超时错误。

## 测试场景

### 场景 1：大型备份导入测试

**测试步骤：**
1. 生成包含大量数据的备份文件
   - 100个站点
   - 每个站点10个账号（共1000个账号）
   - 每个账号5个令牌（共5000个令牌）
   - 50个路由
   - 每个路由20个通道（共1000个通道）

2. 通过管理后台导入该备份文件
3. 监控导入过程，确保：
   - 前端请求不会超时
   - 服务器端处理不会中断
   - 后台任务能够正常完成
   - 数据能够正确导入

**预期结果：**
- 导入过程应在5分钟内完成（前端超时设置为5分钟）
- 所有数据应正确导入
- 系统应正常响应其他请求

### 场景 2：WebDAV 备份导入测试

**测试步骤：**
1. 将大型备份文件上传到WebDAV服务器
2. 配置WebDAV备份设置
3. 触发从WebDAV导入备份
4. 监控导入过程

**预期结果：**
- 导入过程应在1分钟内完成（前端超时设置为1分钟）
- 数据应正确导入
- 系统应记录导入操作的状态

### 场景 3：连续大型操作测试

**测试步骤：**
1. 导入大型备份文件
2. 导出备份
3. 再次导入不同的大型备份文件
4. 监控整个过程

**预期结果：**
- 所有操作应成功完成
- 系统资源使用应合理
- 系统应保持响应

## 技术分析

### 前端超时设置

- `importBackup`: 300,000ms (5分钟) - 足够处理大型数据集
- `importBackupFromWebdav`: 60,000ms (1分钟) - 适合网络传输时间

### 服务器端实现

1. **后台任务系统**:
   - 无内置超时限制
   - 任务状态跟踪
   - 任务完成通知

2. **备份导入逻辑**:
   - 批量插入数据，减少数据库操作次数
   - 使用事务确保数据一致性
   - 分步骤处理不同类型的数据

3. **数据库操作**:
   - 批量插入提高性能
   - 事务确保数据完整性
   - 适合处理大型数据集
